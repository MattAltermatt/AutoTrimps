# CI Deploy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the manual local build+deploy with a GitHub Actions pipeline that builds on push to `main` and publishes one dual-mode userscript to GitHub Pages at a stable URL.

**Architecture:** Actions-native Pages (`actions/upload-pages-artifact` + `actions/deploy-pages`, Pages Source = "GitHub Actions" — no `gh-pages` branch). The build script gains auto-update header metadata, a monotonic CI version, and a landing page. Default branch flips to `main`. No runtime/game-logic changes — only header metadata + delivery pipeline.

**Tech Stack:** Node + esbuild build script (`scripts/build-userscript.mjs`), vitest, GitHub Actions, GitHub Pages.

## Global Constraints

- **No automation-logic or game-balance changes.** Infra only. The emitted JS body must stay byte-identical to a local `npm run build` except for the header metadata (`@version`, `@downloadURL`, `@updateURL`).
- **Stable published URL:** `https://mattaltermatt.github.io/AutoTrimps/autotrimps.user.js` — referenced verbatim in header metadata, landing page, and README. Do not vary it.
- **Gates block deploy:** `npm run typecheck` and `npm test` must pass before build/upload; a failure aborts deployment.
- **Commit messages:** terse one-line subject, no trailers, no `Co-Authored-By`.
- **`@version` must be monotonic** so Tampermonkey detects updates: base `package.json` version locally, `<pkg.version>.<GITHUB_RUN_NUMBER>` in CI.
- **`dist/` stays gitignored** — never commit the build artifact to git (Approach A uploads it as a Pages artifact instead).

### Task 3: GitHub Actions deploy workflow

**Files:**
- Create: `.github/workflows/deploy.yml`

**Interfaces:**
- Consumes: `npm ci`, `npm run typecheck`, `npm test`, `npm run build` (from `package.json` scripts); `dist/` output from the build.
- Produces: a Pages deployment of `dist/` on every push to `main`.

- [ ] **Step 1: Create the workflow**

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy userscript to Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
      - run: npm ci
      - run: npm run typecheck
      - run: npm test
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

- [ ] **Step 2: Sanity-check the YAML parses**

Run: `node -e "const f=require('fs').readFileSync('.github/workflows/deploy.yml','utf8'); if(!/upload-pages-artifact/.test(f)||!/deploy-pages/.test(f)) throw new Error('missing actions'); console.log('workflow OK')"`
Expected: prints `workflow OK`. (Full validation happens on the real CI run in Task 6.)

- [ ] **Step 3: Commit**

```bash

git commit -m "feat(#33): add Actions-native Pages deploy workflow"
```
